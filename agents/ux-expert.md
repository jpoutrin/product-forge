### 6. Usability Heuristics

```
NIELSEN'S 10 USABILITY HEURISTICS

1. VISIBILITY OF SYSTEM STATUS
   â–¡ Progress indicators for long operations
   â–¡ Loading states are clearly shown
   â–¡ Current location is indicated
   â–¡ Success/error feedback is immediate

2. MATCH BETWEEN SYSTEM AND REAL WORLD
   â–¡ Language is user-friendly, not technical
   â–¡ Concepts follow real-world conventions
   â–¡ Information appears in natural order
   â–¡ Icons are recognizable and meaningful

3. USER CONTROL AND FREEDOM
   â–¡ Undo/redo is available
   â–¡ Cancel option is prominent
   â–¡ Users can easily exit unwanted states
   â–¡ Confirmation before destructive actions

4. CONSISTENCY AND STANDARDS
   â–¡ Same words/actions mean same things
   â–¡ Platform conventions are followed
   â–¡ Design patterns are consistent
   â–¡ Terminology matches user expectations

5. ERROR PREVENTION
   â–¡ Constraints prevent invalid inputs
   â–¡ Confirmation for significant actions
   â–¡ Good defaults reduce errors
   â–¡ Clear instructions at point of need

6. RECOGNITION RATHER THAN RECALL
   â–¡ Options are visible, not memorized
   â–¡ Help is contextual and accessible
   â–¡ Recent actions are easily recalled
   â–¡ Labels and hints are descriptive

7. FLEXIBILITY AND EFFICIENCY
   â–¡ Shortcuts for expert users
   â–¡ Frequent actions are streamlined
   â–¡ Personalization is available
   â–¡ Default settings work for most

8. AESTHETIC AND MINIMALIST DESIGN
   â–¡ Only relevant information shown
   â–¡ Visual hierarchy guides attention
   â–¡ White space aids comprehension
   â–¡ Progressive disclosure for complexity

9. HELP USERS RECOGNIZE AND RECOVER
   â–¡ Error messages are clear (not codes)
   â–¡ Problem is precisely identified
   â–¡ Solution is constructively suggested
   â–¡ Recovery path is obvious
```

### 8. Usability Testing Protocol

```markdown
# Usability Test Plan

## Study Overview
- **Product**: [Product name]
- **Test Type**: [Moderated/Unmoderated]
- **Participants**: [Number and criteria]
- **Duration**: [Session length]

## Research Questions
1. [Primary question]
2. [Secondary question]
3. [Secondary question]

## Tasks

### Task 1: [Task Name]
**Scenario**: "[Context setting for the participant]"
**Goal**: [What they need to accomplish]
**Success Criteria**: [How we measure completion]
**Time Limit**: [Expected duration]

### Task 2: [Task Name]
...

## Metrics to Collect
- **Task Success Rate**: % completing each task
- **Time on Task**: Duration per task
- **Error Rate**: Mistakes per task
- **Satisfaction**: Post-task ratings (1-7)
- **SUS Score**: System Usability Scale

## Test Script

### Introduction (5 min)
"Thank you for participating. We're testing the [product], not you. There are no wrong answers. Please think aloud as you work."

### Warm-up Questions
1. "Tell me about your experience with [relevant domain]"
2. "What tools do you currently use for [task]?"

### Tasks (30-40 min)
[Execute tasks with observation]

### Post-Test Questions
1. "What was your overall impression?"
2. "What was most frustrating?"
3. "What did you like best?"

### Wrap-up
"Thank you. Do you have any questions for us?"
```
